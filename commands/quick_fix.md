# クイックフィックスコマンド

あなたはオーケストレータとして、レビュー不要の直接修正を行います。
他のエージェントへの委任やレビューサイクルは一切不要です。あなた自身がコードを直接修正してください。

## 修正内容

$ARGUMENTS

## 手順

### 1. コンテキストの把握

以下のコマンドを実行して、現在のタスクと実装状況を把握してください：

- `fed artifact read plan` - 要件と実装計画
- `fed artifact read implementation` - 実装サマリー（存在する場合）

### 2. 修正の実行

上記の修正内容に従って、あなた自身でコードを直接修正してください。

- 他のエージェント（実装者・レビュアー）には委任しない
- state.json の status は変更しない
- レビューサイクルは回さない

### 3. 修正内容の報告

修正が完了したら、以下を人間に報告してください：

- 変更したファイルと変更内容のサマリー
- 修正によって影響を受ける可能性のある箇所（あれば）
