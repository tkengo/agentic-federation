## コード内に計画・PRD の識別子を残さない

計画（plan アーティファクト）や req-spec が作成した PRD には、ドキュメントを整理するための識別子が付いている（フェーズ番号・作業項目番号・テストシナリオ参照タグ・要件 ID など）。これらは**ドキュメント内での参照専用**であり、コードを後から読む人にとっては何を指すのか分からない。**プロダクションコードにもテストコードにも、これらの識別子を一切残してはならない。** コメント・docstring・テスト名・テスト記述（describe / it / context）・変数名のいずれにおいても禁止する。

### 残してはいけない識別子の例

- フェーズ番号: `Phase 1`、`Phase 2`
- 作業項目番号: `1.1`、`2.3`
- テストシナリオ参照タグ・通し番号: `[1.1]`、`#3`
- PRD の要件・ユーザーストーリー ID: `FR-7`、`US-3` など

### 良くない例

- コメントに `# Phase 2.1: バリデーションを追加 (FR-7)` のように番号を書く
- テスト名に `it('[1.2] should return error when input is empty')` のように参照タグを埋め込む

### あるべき姿

- コメントやテスト名には「コードが何を・なぜするか」だけを書く（例: `it('should return error when input is empty')`）
- 番号を取り除いても意味が通る。通らないなら、そのコメント自体が不要
- 計画やテストシナリオとの対応関係を残したい場合は、コードではなく実装サマリー（artifact）側に書く
